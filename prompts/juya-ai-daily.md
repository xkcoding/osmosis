## 任务

产出 3 块 curator 推送：

### 1. 🔥 核心发布（3-5 条，**每条厚一点**）

从 **要闻 + 模型发布** 挑。**不要只给一行 benchmark**。从该条对应的 `## <标题>` H2 section 的详细段落中榨取 **3-5 个 hard point**，涵盖（有几项写几项，不凑数不编造）：

- benchmark 分数（SWE-Bench / GDPval-AA / TerminalBench 等）
- 参数量 / 架构（总参 / 激活参 / MoE / 稠密）
- 定价 / 上下文 / 上线范围（API 名、三大云平台、开源协议）
- 对标模型或版本差异（vs Opus 4.6 / vs Claude Mythos）
- **quirk**（容易被忽略但值得注意的点）：比如"训练时刻意弱化网络安全"、"tokenizer 改动导致实质涨价"、"xhigh 推理新档位"、"永久上调速率作为补偿"

格式：

**<产品/模型名> · <机构>**
- 事实：一句话（源 blockquote 即可）
- <3-5 个 hard point，每点一个 bullet，含具体数字 / 版本号 / 机制名>
- **对开发者**：一句话「如果你在做 X / 按 Y 计费 / 用 Z 工具链，这意味着 ...」—— 来自源里直接表达或可直接推出的影响，**不脑补因果**
- [原文 ↗](链接) `#N`

### 2. 🛠️ 值得一瞥（3-5 条）

从 **开发生态 + 产品应用** 挑对开发者有直接行动影响的（新增 CLI 命令、配置项、Agent 能力扩展、模型/插件集成数、定价变化）。每条一行，包含具体变化 + 链接 + `#N`。

### 3. 📊 行业信号（0-2 条，可选）

从 **行业动态 + 技术与洞察** 挑带具体数字 / 战略意义的（数据中心规模、政企合作、融资金额、安全机制），**剔除**：前瞻与传闻、含"传 / 计划 / 拟 / 或将 / 据悉"字眼的条目、单纯公司动态无具体数字。

## 硬约束（违反任意一条 = 重写）

### 不加戏
- 不脑补因果链、不升级措辞、不加修辞补语（"已形成" / "引发激烈讨论" / "前所未有" / "标志性" / "击败" / "碾压"）
- 原文若无基准分 / 参数量 / 价格，不要自造；没 hard point 就写不到 5 个，不凑数
- `前瞻与传闻` 整个类别跳过；含 "传" / "计划" / "拟" / "或将" / "据悉" 字眼的条目也跳过
- 产品名、机构名、模型 ID 严格照搬原文大小写和版本号（Claude Opus 4.7，不是 Claude 4.7；GPT-Rosalind，不是 GPT Rosalind）
- **「对开发者」一行**只能从源中已有事实推出的直接影响写，不做战略揣测

### 能读
- 每句 ≤ 35 字；长就断句或改 bullet
- 每节开头 1 个 emoji 就够
- 核心发布的 hard point 用同一 bullet 层级，不嵌套

### 形态
- 每条保留原文链接（markdown `[...](链接)`）和 `#N` 序号
- 中文输出
- **不设字数上限**；但每节保持密度，不用字数堆质感
- 输出直接从 `**🔥 核心发布**` 开头
